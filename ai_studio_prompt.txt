Develop an interface that has two sections.
The title of the interface is "ATMA - Alzheimer's & Dementia Triage Monitoring System."
Section A named as baseline contains of three sub sections that has button to allow to upload an image, audio and json file.
Once the data is uploaded, a done check mark should be displayed. In the backend the baseline uploaded data along with the system prompt should be prepared in a message list.
Section B named as real time contains of three sub sections that has button to allow to upload an image, audio and json file.
Once the data is uploaded, the data should be appended to the message list.
The message list should be sent to gemini 3 for analysis. The system prompt should be as follows as quoted in <system_prompt> </system_prompt> tags.

SENSOR DATA: A short text summary of objective vitals or environmental data (e.g., Heart Rate, Temperature, Door Status).

CRISIS (Level 1): Trigger if any of the following combinations are present:
Evidence of Fall (Image: Patient on floor, abnormal posture) AND Evidence of Acute Distress (Voice: Screaming, sudden crying, slurring, or groaning).
Image analysis confirms Visible Blood or Head Trauma AND Sensor data shows Elevated Heart Rate (>120 BPM).
Sensor Data indicates Wandering/Hazard (e.g., "Front door open at 3 AM") AND Voice transcript contains Paranoid or Panicked Speech.

Image shows Agitation/Pacing (e.g., restless movement) AND Voice transcript shows Severe Confusion or Verbal Aggression.
